Implementation Spec

Authoritative reference for architecture, technology decisions, data model, API, and the NixOS deployment module. Keep this document in sync with the code.

Goals

  • Any authenticated user pastes a URL, picks a category, hits go — file lands in the correct Jellyfin subdirectory with proper folder structure, NFO metadata, and poster art.
  • Admin users manage categories (name → filesystem path) via the web UI.
  • Full download queue visible to all users; history of past downloads.
  • MP4 output; per-download resolution picker (auto-detected from the URL, defaults to 1080p).
  • yt-dlp archive file per category to prevent re-downloading.
  • Works with any yt-dlp-supported site, not just YouTube.

Non-Goals

  • No Jellyfin API integration — rely on Jellyfin's scheduled library scan.
  • No channel/playlist subscription (that's Pinchflat's job).
  • No concurrent downloads — serial queue is intentional (avoids rate limits).

Output File Structure

{category_path}/
└── {display_title} [{video_id}]/
    ├── {display_title} [{video_id}].mp4
    ├── {display_title} [{video_id}].nfo
    └── {display_title} [{video_id}]-poster.jpg

display_title is the user-supplied custom title if provided, otherwise the yt-dlp title. The [video_id] suffix is always appended. File/folder names are sanitized via yt_dlp.utils.sanitize_filename().

Staging

Downloads are assembled in {STAGING_DIR}/{download_id}/ and atomically moved to the final location only when complete. On app startup, any leftover staging subdirectories from a previous crash are deleted and interrupted downloads are reset to pending before the worker re-queues them.

Filesystem note: For atomic moves (rename, not copy+delete), STAGING_DIR must be on the same mount as the Jellyfin library. Override the NixOS module default (/var/lib/ytdlfin/staging) to a path on the same filesystem as your media library, e.g. /orico/jellyfin/.ytdlfin-staging.

Database

SQLite at {DATA_DIR}/ytdlfin.db. Default journal mode (DELETE, not WAL) — no sidecar files, safe for restic snapshots between transactions.

Authentication

PocketID OIDC via authlib.integrations.starlette_client. Session stored client-side in a signed cookie (itsdangerous / SessionMiddleware). Session contains: sub, email, name, is_admin.

Access control is group-based. The OIDC token is requested with the groups scope; PocketID returns a groups claim (array of group name strings) in the userinfo response. Two groups are configured via env vars:

  • ADMIN_GROUP — members receive admin privileges (category management, full history visibility)
  • USER_GROUP — members receive regular user access (submit downloads, view own history)

A user not in either group is redirected to /auth/denied after a successful PocketID login. Their session is cleared immediately. No per-request DB lookup is needed — group membership is resolved once at login and stored in the session.

Reverse Proxy

ytdlfin is designed to run behind nginx with TLS termination. Two things must be configured correctly for the OIDC flow and session cookies to work:

nginx configuration

nginx must forward the real client IP and scheme so uvicorn can reconstruct the correct request URL:

location / {
    proxy_pass http://127.0.0.1:8001;
    proxy_set_header Host              $host;
    proxy_set_header X-Real-IP         $remote_addr;
    proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

Application behaviour

When HTTPS_ONLY=true (set automatically by the NixOS module):

  • uvicorn trusts X-Forwarded-Proto from upstream proxies, so request.url.scheme returns https. This is required for authlib to construct the correct OIDC redirect URI.
  • SessionMiddleware marks cookies Secure, so browsers only transmit the session over TLS.

TRUSTED_PROXY_IPS controls which upstream IPs are allowed to set forwarded headers. The default 127.0.0.1 is correct for same-host nginx. Set to * to trust all upstream proxies, which is safe for LAN deployments protected by a physical firewall where nginx may be on a different host.

Dev note: Leave HTTPS_ONLY unset (defaults to false) for local HTTP development. The session cookie will not be marked Secure, which is correct for plain HTTP.

Decision Log

Serial download queue (no concurrency)
Intentional — avoids rate limits. A single asyncio.Queue processed by one background task keeps the implementation simple and the load on the source site low.
Manual queue start
Downloads do not begin automatically after being queued. The user clicks "Start downloads" which calls POST /api/queue/start, enqueuing all pending items for the worker. This gives users a chance to review and reorder before committing.
SQLite default journal mode (not WAL)
Single writer means WAL offers no benefit. Default mode produces no sidecar files, so restic can snapshot the DB file at any time without extra configuration.
Staging + atomic move
Downloads are assembled in a staging directory and renamed into the final location only when all files (MP4, NFO, poster) are ready. This prevents Jellyfin from picking up partial downloads during its library scan.
yt-dlp as Python library, not subprocess
Avoids subprocess overhead and makes progress hooks and error introspection easier. The yt-dlp binary is still available in the dev shell for manual testing.
No migrations framework
CREATE TABLE IF NOT EXISTS on startup is sufficient for v1. If the schema needs to change, add a manual migration step in the release notes.
Group-based access control (not email allowlist)
Access is gated by PocketID group membership (ADMIN_GROUP / USER_GROUP) rather than a hardcoded email list. Groups are managed in PocketID and resolved at login via the groups OIDC scope; the result is stored in the session so no per-request lookups are needed. Fails closed: a user not in either group is denied even after a successful PocketID login.
CDN with SRI hashes (Pico CSS, HTMX)
No build step. Both resources are pinned to exact versions with SHA-384 integrity hashes computed from the actual CDN content. Update the hashes whenever versions change.
Reverse proxy headers trusted from 127.0.0.1 by default
The default TRUSTED_PROXY_IPS=127.0.0.1 is correct for the standard deployment where nginx runs on the same host as ytdlfin. For LAN deployments where nginx is on a separate host behind a physical firewall that controls perimeter access, set TRUSTED_PROXY_IPS=* (or services.ytdlfin.settings.trustedProxyIps = "*") to trust all upstream proxies.
